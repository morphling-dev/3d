# Contributing to Morphling 3D

We welcome and appreciate contributions from the community! Whether you want to fix bugs, add new features, improve documentation, or help with testing, your input helps make Morphling 3D better for everyone.

## Ways to Contribute

- **Open Issues:**  
  Report bugs, suggest new features, or request improvements using our [GitHub issue tracker](https://github.com/morphling-dev/3d/issues).

- **Submit Pull Requests:**  
  If you have improvements, bug fixes, or new generators you’d like to contribute, fork the repository and create a pull request against the `main` branch. Please include a clear description of your changes.

- **Improve Documentation:**  
  Typos, missing details, or clearer instructions are always welcome! You can contribute directly to `README.md` or open a documentation-specific issue.

- **Community Support:**  
  Help answer questions or share tips with other users in issues or discussions.

## Guidelines

- Follow the general coding standards and conventions of the codebase.
- Make sure new features or stubs are relevant to Morphling 3D’s core mission: simplifying DDD and modular Laravel development.
- Write clear, descriptive commit messages.
- Add tests if you introduce critical new logic or bug fixes.
- Keep pull requests focused and as small as possible.
- Be respectful and considerate in all interactions.

## How to Get Started

1. **Fork the repository** to your own GitHub account.
2. **Clone your fork**:
   ```bash
   git clone https://github.com/<your-username>/3d.git
   ```
3. **Create a feature branch**:
   ```bash
   git checkout -b your-feature-name
   ```
4. **Make your changes.**
5. **Test your changes locally.**
6. **Push to your branch**:
   ```bash
   git push origin your-feature-name
   ```
7. **Create a Pull Request** from your branch to the `master` branch of the official repo.

## Code of Conduct

Help us keep this project welcoming! Please review and follow our [Code of Conduct](https://github.com/morphling-dev/3d/blob/main/CODE_OF_CONDUCT.md).

## Questions?

If you’re unsure how to contribute, feel free to open an issue or start a discussion. We’re happy to guide you!
