# Architecture Overview

Morphling 3D transforms your Laravel project into a high-performance **Domain-Driven Design (DDD)** environment. By shifting from a "feature-by-file-type" (MVC) to a "feature-by-domain" (Modular) approach, your codebase remains organized even as it grows to hundreds of modules.

-----

## The 4-Layer Blueprint

Every module generated by Morphling 3D follows a strict, four-layer hierarchy. This separation ensures that your core business logic (The Domain) remains untouched by framework updates or database migrations.

```text
modules/{ModuleName}/
├── Delivery/           # The "Entry Point" (HTTP, CLI, UI)
├── Application/        # The "Orchestrator" (Use Cases, DTOs)
├── Domain/             # The "Brain" (Entities, Rules, Interfaces)
└── Infrastructure/     # The "Worker" (Eloquent, APIs, Repositories)
```

-----

## Automated Scaffolding: What `module:new` Builds

When you run `php artisan module:new {Name}`, Morphling 3D doesn't just create folders; it generates a fully-wired execution chain.

### The Delivery Layer

  * **Controller:** Maps HTTP verbs to Use Cases.
  * **Routes:** Auto-registered API/Web endpoints.
  * **FormRequest:** Self-contained input validation.

### The Application Layer

  * **UseCase:** A single-purpose class (e.g., `CreateOrderUseCase`) that coordinates the flow.
  * **DTO (Data Transfer Object):** A type-safe contract that carries data between layers.

### The Domain Layer

  * **Entity:** A plain PHP class representing a business concept (e.g., `User`).
  * **Repository Interface:** A contract defining how to fetch/save data without mentioning SQL.

### The Infrastructure Layer

  * **Eloquent Model:** The standard Laravel model used for database interaction.
  * **Repository Implementation:** The concrete logic that fulfills the Domain Interface using Eloquent.

-----

## Core Advantages

### 1\. Isolation of Concerns

Because the **Domain** doesn't know about the **Infrastructure**, you can swap your database from MySQL to MongoDB or replace an Eloquent model with an external API call without changing a single line of business logic.

### 2\. Auto-Discovery

Morphling 3D eliminates the "Registration Tax." The engine automatically scans your `modules/` folder and registers:

  * Service Providers
  * Routes (Web & API)
  * Blade View Namespaces
  * Database Migrations

### 3\. Developer Experience (DX)

Generated views include **Deep-Linking** capabilities. When viewing a module in your browser, the `index.blade.php` file provides direct links to open the specific Controller or Use Case in **VSCode, Cursor, or PHPStorm**.
