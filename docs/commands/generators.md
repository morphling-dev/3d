# Artisan Generators: Command Cheat Sheet

Morphling 3D provides a robust set of Artisan commands that scaffold code according to strict DDD and Hexagonal conventions. Instead of manually creating directories or editing namespaces, leverage these generators to ensure every component lands in the correct layer and structure.

---

## Core Generation Commands

The main entry point for a new module is:

```bash
php artisan 3d:new {ModuleName}
```
**What it does:** Instantly scaffolds a fresh, ready-to-work, 4-layer module. All subfolders and base provider classes are created—no manual setup needed.

## Anatomy of a Generated DTO

DTOs generated by Morphling 3D are immutable (`readonly`). This guarantees that once data is validated (typically in a Request), it cannot be tampered with across the application.

```php
readonly class UserDto
{
    public function __construct(
        public array $data
    ) {}

    /**
     * Morphling auto-creates this bridge: converts validated request data
     * into a strongly-typed DTO instantly.
     */
    public static function fromRequest(mixed $request): static
    {
        return new static($request->validated());
    }
}
```

## Best-Practice Workflow Example

To build a new feature (e.g., "User Registration") inside an existing module, use this ideal order:

1.  **Domain:** `3d:make-entity`, `3d:make-interface`  
    (Define core objects and business contracts.)
2.  **Infrastructure:** `3d:make-model`, `3d:make-repo`  
    (Persistence and technical implementation.)
3.  **Application:** `3d:make-dto`, `3d:make-usecase`  
    (Input/output flow and orchestration.)
4.  **Delivery:** `3d:make-request`, `3d:make-controller`  
    (HTTP interface—entry and exit to the outside world.)

## Generator Tips & Caveats

> [!TIP]
> **No Duplicate Suffixes Needed:**  
> Run `php artisan 3d:make-usecase RegisterUser {Module}` (not `RegisterUserUseCase`).  
> Morphling adds the "UseCase", "Dto", etc. suffixes automatically for you.

> [!WARNING]
> **Namespace Management:**  
> If you manually move any generated file, update its namespace. Morphling sets namespaces based on directory structure under `modules/`.

## Troubleshooting

### Command not found
Double-check you ran `php artisan 3d:install`. This installs and registers the Morphling service provider.  
Also confirm `Modules\\` is registered in your `composer.json` autoload section.

### Stub file missing
To customize templates, publish the stubs with:
```bash
php artisan vendor:publish --tag=morphling-stubs
```
Edit your published stubs in the root `stubs/` directory.
