# **Integration Tests (Infrastructure Layer)**

While Unit and Mock tests verify our logic, **Integration Tests** verify that our code actually talks to the outside world correctly. In Morphling 3D, this specifically means testing your **Eloquent Repositories**, **Mappers**, and **Database Migrations**.

---

## **The Goal of Integration Testing**
We are testing the "Plumbing." We want to ensure that:
1.  The SQL generated by Eloquent is correct.
2.  The Database schema (Migrations) supports our data needs.
3.  The **Mapper** correctly converts a Database Row into a **Domain Entity** and back.



---

## **1. Setting Up the Database Environment**
Unlike Unit tests, Integration tests **must** boot Laravel and connect to a database. To keep them fast, we typically use **SQLite in-memory**.

```php
namespace Modules\Transaction\Tests\Integration\Infrastructure\Repositories;

use Tests\TestCase; // Use Laravel's base TestCase
use Illuminate\Foundation\Testing\RefreshDatabase;
use Modules\Transaction\Infrastructure\Models\TransactionModel;
use Modules\Transaction\Infrastructure\Repositories\EloquentTransactionRepository;
use Modules\Transaction\Domain\Entities\TransactionEntity;

class EloquentTransactionRepositoryTest extends TestCase
{
    use RefreshDatabase; // Wipes the DB for every test

    protected EloquentTransactionRepository $repository;

    protected function setUp(): void
    {
        parent::setUp();
        // Instantiate the real concrete implementation
        $this->repository = new EloquentTransactionRepository(new TransactionModel());
    }
}
```

---

## **2. Testing "Save" and "Find"**
We want to verify that an Entity saved via the Repository can be retrieved exactly as it was.

```php
public function test_it_can_save_and_retrieve_a_transaction_entity()
{
    // 1. Create a pure Domain Entity
    $entity = new TransactionEntity(id: 'trx_100', amount: 1500, status: 'pending');

    // 2. Save it using the Repository
    $this->repository->save($entity);

    // 3. Retrieve it from the DB
    $retrieved = $this->repository->findById('trx_100');

    // 4. Assertions
    $this->assertInstanceOf(TransactionEntity::class, $retrieved);
    $this->assertEquals(1500, $retrieved->getAmount());
    $this->assertEquals('pending', $retrieved->getStatus());
}
```

---

## **3. Testing Mappers (Translation Logic)**
If your Repository uses a separate **Mapper** class, you should test that complex data (like JSON columns or Date formats) are translated correctly between the DB and the Domain.

```php
public function test_mapper_converts_database_json_to_value_objects()
{
    $model = TransactionModel::factory()->create([
        'metadata' => json_encode(['ip' => '127.0.0.1', 'device' => 'mobile'])
    ]);

    $entity = $this->repository->findById($model->id);

    // Verify the internal Entity properties match the DB JSON
    $this->assertEquals('127.0.0.1', $entity->getMetadata()->ip);
}
```

---

## **Why This is Critical**

* **Catching SQL Errors:** It ensures your migrations have the right column names and types.
* **Mapper Validation:** It prevents "Data Leakage" where a database null value might crash a Domain Entity that expects a string.
* **Confidence:** This is the final check that your "Clean Architecture" actually works when it hits the disk.

---

## **Best Practices**

* **Keep it Focused:** Don't test business logic here; only test that data goes in and comes out correctly.
* **Avoid External APIs:** If your Infrastructure talks to a 3rd party API (like Stripe or AWS), use **Mocks** or **Http::fake()** even in integration tests to avoid slow network calls.
* **Use Factories:** Use Laravel's Model Factories to quickly set up the database state before running your Repository methods.
